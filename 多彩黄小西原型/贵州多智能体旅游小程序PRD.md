# 一、PRD文档（多彩黄小西小程序 - 高保真原型版）

## 1. 产品概述

### 1.1 产品名称
多彩黄小西小程序

### 1.2 核心定位
以多智能体为核心，聚焦贵州旅游全场景，通过“对话即服务”的交互方式，为用户提供一站式、个性化、沉浸式的旅游服务体验。

### 1.3 视觉风格
- **主色调**：苗银灰 (#5E6E82) —— 沉稳、高级，体现贵州非遗文化。
- **强调色**：刺绣红 (#FF7157) —— 活力、热情，用于核心操作和视觉焦点。
- **辅助色**：蜡染蓝 (#2C5282) —— 搭配使用，丰富视觉层次。
- **背景**：冷灰白基调 (#F5F7FA) 叠加贵州特色纹样（苗银/蜡染纹理）的线性渐变，营造地域氛围。

## 2. 核心功能模块（基于 index.html 原型）

### 2.1 全局导航与布局

#### 2.1.1 微信小程序顶部导航 (Custom Navigation Bar)
- **样式**：自定义导航栏，背景全透明，内容沉浸式展示。
- **元素**：
  - **标题**：“多彩黄小西” (黑色，字重600，居中)。
  - **胶囊按钮**：右侧悬浮胶囊（... | ×），模拟微信原生样式，半透明磨砂玻璃质感。
- **交互**：固定在页面顶部，不随页面滚动。

#### 2.1.2 底部导航栏 (Bottom TabBar)
- **样式**：白色半透明磨砂背景，顶部细边框。
- **结构**：5个 Tab 项。
  1.  **首页**：图标（选中态为浅蓝色圆形背景 #E6F7FF，高亮显示）。
  2.  **行程**：图标。
  3.  **发布 (+)**：中心突出设计，深色圆形背景 (#1A202C)，白色图标，用于快速创建/发布行程。
  4.  **优选**：图标。
  5.  **我的**：图标。
- **交互**：点击切换模块，点击中间“+”号唤起发布菜单。

---

### 2.2 智能体广场（首页核心区）

#### 2.2.1 顶部英雄区 (Hero Section)
- **功能行**：
  - **公告胶囊**：左侧展示最新资讯（如“贵州文旅优惠季开启...”），支持点击查看更多。
  - **快捷操作**：右侧圆形按钮组（语言切换、消息通知）。
- **IP 形象展示**：
  - **人物**：展示“黄小西”IP形象（苗族服饰少女），位于右侧，作为智能助手化身。
  - **气泡对话**：左侧悬浮气泡，动态展示欢迎语（如“你好，我是黄小西，欢迎来到多彩贵州”），带浮动动画效果。

#### 2.2.2 智能体分类 (Category Tabs)
- **样式**：纯文字胶囊按钮，横向滚动。
- **分类**：
  1.  **旅游小助手**（默认选中）：行程规划、旅居设计、AI伴游等。
  2.  **酒店**：贵州饭店、特色民宿等。
  3.  **景区**：黄果树、小七孔、梵净山等。
  4.  **个人**：地陪导游、旅拍摄影师等。
- **交互**：点击 Tab 切换下方智能体列表，选中项背景变为苗银灰 (#5E6E82)，文字变白。

#### 2.2.3 智能体列表 (Agent Grid)
- **布局**：双列瀑布流/网格布局，纵向滚动。
- **卡片样式**：
  - **背景**：全卡片背景图（实景图片，如酒店房间、景区风景），覆盖半透明渐变蒙层以保证文字可读性。
  - **无头像**：移除传统的圆形头像，强调沉浸式视觉体验。
  - **内容**：底部展示智能体名称（如“旅游规划”、“贵州饭店·贵宾楼”）、简短描述、功能标签（如“行程定制”、“高端住宿”）。
- **交互**：点击卡片直接进入该智能体的对话界面。

#### 2.2.4 底部悬浮对话框 (Floating Chat Input)
- **位置**：位于智能体列表下方，底部导航栏上方，跟随页面流布局（非固定遮挡）。
- **样式**：
  - 浅色半透明磨砂背景（Glassmorphism），圆角矩形（Pill Shape）。
  - **左侧**：白色圆形语音按钮（麦克风图标）。
  - **中间**：输入框，默认占位符“夜景游玩路线怎么安排?”。
  - **右侧**：白色圆形发送按钮（纸飞机图标）。
- **功能**：作为全局主智能体入口，用户无需选择具体智能体，直接输入问题即可获得综合解答。

## 3. 交互与体验细节

### 3.1 滚动与层级
- **页面结构**：Flex Column 布局。
  - **Top**：固定 Header + 英雄区 + 分类 Tab。
  - **Middle**：可滚动的智能体列表区域 (`.agent-grid-scroll`)。
  - **Bottom**：对话输入框 + 底部导航栏。
- **防遮挡处理**：内容区域底部留白 (`padding-bottom`) 动态计算，确保最后一张卡片能完整滑出，不被底部导航栏遮挡。

### 3.2 视觉反馈
- **点击态**：按钮、卡片点击时有轻微缩放或透明度变化。
- **选中态**：底部导航图标增加浅蓝色背景光环，强化当前状态感知。
- **动态效果**：气泡浮动动画 (`@keyframes float`)，增加页面灵动感。

## 4. 技术实现要求 (Web/H5)

- **适配性**：
  - 必须适配 iOS 底部安全区域 (`safe-area-inset-bottom`) 和顶部刘海屏 (`safe-area-inset-top`)。
  - 使用 `calc()` 动态计算高度和间距。
- **资源加载**：
  - 图片资源需使用 CDN 或本地优化后的资源。
  - 字体优先使用系统默认字体栈（PingFang SC / San Francisco）。
- **CSS 特性**：
  - 大量使用 `backdrop-filter: blur()` 实现毛玻璃效果（Header, Nav, Chat Input）。
  - 使用 CSS Grid 实现智能体卡片布局。
